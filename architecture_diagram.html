High-Level System Architecture

The College Event Portal is a client-side Single Page Application (SPA) built with React 19 and TypeScript. It uses a dual-storage strategy with localStorage for instant offline access and Supabase Cloud for cross-device persistence. The app is built with Vite 6 and deployed as static assets on Vercel's Edge CDN.

💾 Dual Storage Strategy

localStorage provides instant offline access. Supabase Cloud enables cross-device data sync. BroadcastChannel handles cross-tab synchronization.

⚡ Build Pipeline

Vite 6 provides lightning-fast HMR in development and optimized bundling for production. TypeScript ensures compile-time type safety.

🌐 Deployment

Static SPA deployed on Vercel Edge CDN for global low-latency delivery. Zero server-side rendering required.

🔒 Security

Environment-based Supabase config via VITE_SUPABASE_URL and VITE_SUPABASE_ANON_KEY. Role-based access control at the application layer.

Frontend Component Hierarchy

The application follows a flat component architecture with App.tsx as the central orchestrator. All 8 feature components are direct children of App, receiving state and callbacks via props. No external state management library is used — React's built-in useState and useEffect hooks handle all state.

Data Flow Architecture

User interactions trigger CRUD operations in the database layer (db.ts), which persists data to localStorage and optionally syncs to Supabase Cloud. State changes propagate back to React via the refreshDbState() pattern, which re-reads localStorage and updates component state through useState hooks.

Key Architecture Decisions

💾 Cloud Sync with Local Fallback

Uses Supabase Cloud combined with localStorage and BroadcastChannel. Multi-device sync with zero-setup friction and full offline support.

⚛️ No External State Library

React's built-in useState/useEffect hooks handle all state. The refreshDbState() pattern re-reads localStorage and triggers re-renders — simple and effective.

👁️ Real-Time Admin Roster

Student signups and logins immediately trigger state reloads, giving admins real-time visibility into student activity, login timestamps, and bookings.

🛡️ Admin Revocation Power

Admins can revoke any student's event registration from the Student Roster table. Cancelled seats are automatically returned to the event's capacity pool.

🎨 Time-of-Day Theme Engine

Auto theme switcher transitions between daytime light themes and nighttime Midnight Neon dark mode based on the user's local time.

📁 Flat Component Architecture

All 8 feature components are direct children of App.tsx. No deep nesting or context providers — state flows through props for simplicity and predictability.
